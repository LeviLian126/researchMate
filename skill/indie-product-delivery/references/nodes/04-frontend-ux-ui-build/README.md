# Frontend Build

Node04 owns user-facing flow, information architecture, content, visual direction, component implementation, responsive and accessible behavior, integration states, and local browser proof. Use it directly for an approved local UI change; do not reopen product or architecture without a real boundary gap.

| Need | Read |
|---|---|
| recover the surface and design its journey, hierarchy, content, and complete states | `experience-flow-content-and-states.md` |
| establish visual direction, translate references/prototypes, or perform a redesign | `visual-direction-prototype-and-redesign.md` |
| build production components, responsiveness, accessibility, and motion | `frontend-responsive-accessible-build.md` |
| verify in a browser and debug rendering or integration failures | `browser-proof-and-visual-debug.md` |
| build or refresh durable HTML project documentation | `references/agent-context-html/instructions.md` |

A copy, token, spacing, or contained component fix normally stays in Node04 with narrow proof. Use Node01 only when the page would decide audience, promise, pricing, positioning, or primary workflow. Use Node02/03 when API, data, permission, provider, state ownership, dependency, or backend behavior is missing.

Treat a design-system choice as distinct from visual language: use one adopted system's official components and tokens when present, and never claim an official system from visual resemblance alone. Apply art-direction controls consistently across the surface, vary macro composition with information relationships, and keep the micro system coherent.

Keep continuous pointer, scroll, and timeline values out of ordinary component state when they would cause persistent rerenders; prefer compositor-friendly properties, clean up listeners/observers/animation instances, and provide reduced-motion behavior. Define a reusable visual-block contract only for genuine cross-page reuse.

Node04 may establish local browser evidence but Node05 owns final quality, security, and ship judgments. Browser absence must be reported rather than replaced by screenshot or source inspection claims.
