定位

这个 skill 负责什么

浏览器类能力给 agent 真实眼睛和手。它们通过持久 Chromium 会话完成点击、截图、登录态测试和跨 agent 协作。

网页数据抽取入口。首次原型化抽取流程，后续可路由到固化 browser-skill。 它在 gstack 中的角色不是“万能提示词”，而是一个清晰的工作边界：什么时候触发、先检查什么、产出什么、哪些事情必须交给其他能力处理。

适用场景

  • 从网页读表格/列表
  • 抽取公开页面数据
  • 需要 JSON 输出

工作流程

  • Step 1. 打开页面
  • Step 2. 定位数据结构
  • Step 3. 抽取并格式化
  • Step 4. 判断是否可固化

输出结果

  • JSON 数据
  • 抽取步骤
  • 可 skillify 线索
边界

避免误用的规则

真实产品文档要讲清楚能力边界。这个页面把 skill 的适用范围和不适用范围放在显眼位置，方便用户判断是否应该调用它。

只读抽取，不处理提交表单等修改动作。

与其他能力的关系

同类能力包括 browse、devex-review、hackernews-frontpage、open-gstack-browser。实际使用时，先由 landing page 或上游 workflow 判断意图，再把任务交给最贴近目标的 skill。
