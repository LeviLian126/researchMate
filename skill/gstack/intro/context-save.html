定位

这个 skill 负责什么

安全与记忆类能力保护工作区，保存上下文，并让工具记住长期偏好和项目经验。

保存当前工作上下文，包括 git 状态、决策、剩余任务和关键线索。 它在 gstack 中的角色不是“万能提示词”，而是一个清晰的工作边界：什么时候触发、先检查什么、产出什么、哪些事情必须交给其他能力处理。

适用场景

  • 中断前保存进度
  • 跨 workspace handoff
  • 长任务需要 checkpoint

工作流程

  • Step 1. 读取 git 状态
  • Step 2. 总结决策
  • Step 3. 记录剩余工作
  • Step 4. 保存上下文

输出结果

  • context snapshot
  • 恢复线索
  • 工作状态
边界

避免误用的规则

真实产品文档要讲清楚能力边界。这个页面把 skill 的适用范围和不适用范围放在显眼位置，方便用户判断是否应该调用它。

只保存工作上下文，不替代正式 commit。

与其他能力的关系

同类能力包括 careful、context-restore、freeze、gstack-upgrade。实际使用时，先由 landing page 或上游 workflow 判断意图，再把任务交给最贴近目标的 skill。

示例触发

/context-save 帮我处理当前项目中最相关的工作流，并在输出里列出证据、风险和下一步。
