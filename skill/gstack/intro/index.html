项目总览

把一次 coding session 变成可审计的工程流程

gstack 面向需要真实交付的软件项目：它用明确的角色、质量门、浏览器证据和发布步骤，把 AI 生成代码纳入可追踪、可验证、可复盘的工作系统。

01

从想法到 PR 的流程

Think → Plan → Build → Review → Test → Ship → Reflect，每一步都有明确输入、输出和质量门。

02

真实浏览器作为证据

长驻 Chromium daemon 保留 cookie、tab 和 localStorage，让 QA、截图、控制台和网络检查基于真实页面状态。

03

发布前后都有护栏

review、cso、qa、ship、land-and-deploy、canary 串起代码审查、部署和生产验证。

FAQ

gstack 如何提升 AI 编程质量

gstack 把模型能力放进工程流程里，让 agent 不只会写代码，也会检查需求、设计体验、验证页面、守住安全边界并完成发布收尾。

gstack 解决的核心问题是什么？

它解决的是 AI 编程缺少流程的问题：需求没压实、计划没评审、页面没真实点击、发布后没验证，都会让代码看起来完成但实际不可用。

为什么需要持久浏览器？

许多产品问题只会在真实页面中暴露。持久浏览器保留登录态、tab、控制台、网络和截图证据，让 agent 能像 QA 工程师一样检查体验。

它如何降低发布风险？

gstack 把 review、security、QA、benchmark、ship、deploy 和 canary 串成闭环，让风险在合入前、部署中和上线后都被检查。
