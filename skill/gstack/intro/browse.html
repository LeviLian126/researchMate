定位

这个 skill 负责什么

浏览器类能力给 agent 真实眼睛和手。它们通过持久 Chromium 会话完成点击、截图、登录态测试和跨 agent 协作。

核心浏览器 skill，提供导航、点击、输入、截图、控制台、网络、响应式和断言能力。 它在 gstack 中的角色不是“万能提示词”，而是一个清晰的工作边界：什么时候触发、先检查什么、产出什么、哪些事情必须交给其他能力处理。

适用场景

  • 测试网页
  • 检查 UI 状态
  • 复现浏览器问题

工作流程

  • Step 1. 启动或连接 daemon
  • Step 2. 导航目标
  • Step 3. 用 refs 操作元素
  • Step 4. 收集截图和日志

输出结果

  • 页面快照
  • 截图
  • 控制台/网络证据
边界

避免误用的规则

真实产品文档要讲清楚能力边界。这个页面把 skill 的适用范围和不适用范围放在显眼位置，方便用户判断是否应该调用它。

浏览器只给证据，业务修复交给 QA 或具体开发流程。

与其他能力的关系

同类能力包括 devex-review、hackernews-frontpage、open-gstack-browser、pair-agent。实际使用时，先由 landing page 或上游 workflow 判断意图，再把任务交给最贴近目标的 skill。

示例触发

/browse 帮我处理当前项目中最相关的工作流，并在输出里列出证据、风险和下一步。
