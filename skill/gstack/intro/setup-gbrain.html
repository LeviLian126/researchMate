定位

这个 skill 负责什么

安全与记忆类能力保护工作区，保存上下文，并让工具记住长期偏好和项目经验。

从零配置 GBrain，让 coding agent 获得持久记忆和 MCP 工具。 它在 gstack 中的角色不是“万能提示词”，而是一个清晰的工作边界：什么时候触发、先检查什么、产出什么、哪些事情必须交给其他能力处理。

适用场景

  • 安装 gbrain
  • 初始化 PGLite/Supabase
  • 设置仓库信任策略

工作流程

  • Step 1. 安装 CLI
  • Step 2. 选择引擎
  • Step 3. 注册 MCP
  • Step 4. 记录 trust policy

输出结果

  • 可用 gbrain
  • MCP 配置
  • 信任策略
边界

避免误用的规则

真实产品文档要讲清楚能力边界。这个页面把 skill 的适用范围和不适用范围放在显眼位置，方便用户判断是否应该调用它。

涉及凭据和隐私时必须用户确认。

与其他能力的关系

同类能力包括 careful、context-restore、context-save、freeze。实际使用时，先由 landing page 或上游 workflow 判断意图，再把任务交给最贴近目标的 skill。

示例触发

/setup-gbrain 帮我处理当前项目中最相关的工作流，并在输出里列出证据、风险和下一步。
