交接记录与协作计划

本计划结合 PROJECT_DEVELOPMENT_PROCESS 与 gstack 从 API Spec 到 Execution Plan 的交付链路。拆分原则是先看上下文边界，再看人数：如果任务共享同一段阻塞上下文，就由同一个 agent 顺序完成，不为了并行而强拆。

判断规则：同一 agent 应连续完成会共同修改同一批 API、表结构、状态机、索引管线或运行时上下文的任务；只有输入输出契约稳定、互不抢同一上下文时，才拆给不同 agent 并行。
执行组负责范围执行方式依赖条件验收标准
Agent A：数据闭环主线 Auth、Project、Storage、Upload、Parser、Index、Local Ask 顺序完成，不拆分。该链路共享用户隔离、文档状态机、R2 对象、chunk 元数据、Qdrant payload 和本地 RAG 运行上下文。 当前 API、DB、OpenAPI、TypeScript 契约稳定。 用户 A 不能读取用户 B 数据；25MB 与 MIME 校验生效；文档可完成上传、解析、切分、入库和本地问答；Local Ask 的 citations 必须来自本地检索 chunk。
Agent B：来源策略与 Web 扩展 Source Policy、Hybrid、Web Search、Crawl、Web Evidence 顺序完成。Mode resolver、tool policy validator、web cache 与 citation 规则共享同一策略上下文，不能拆成多个 agent 抢改。 Agent A 已交付 Local Ask 闭环。 Local mode 阻止 web 工具；Hybrid 遵循 local-first；失败网页不得进入 citations；web evidence 有来源、时间和缓存状态。
Agent C：Quiz 输出链路 Quiz generation、Quiz save、Quiz display contract 可在 Agent A 完成 Parser + Index 后独立进行；不依赖 Web 扩展。 文档 chunk、引用字段和项目归属校验稳定。 选择题正好 4 个选项；答案、解释、引用来源都通过 schema 校验；普通用户只能看到自己的 quiz set。
Agent D：前端 MVP Ask、Library、Quiz 页面与前端状态管理 可先基于契约开发壳层；最终联调必须等 Agent A 的认证、上传和本地问答接口稳定后顺序完成。 API contract、shared TypeScript contracts、核心接口返回形状稳定。 Ask、Library、Quiz 可完成主流程；普通用户界面没有 Trace 入口；错误态与加载态不泄露内部 trace 或 secret。
Agent E：Trace 与 Eval Developer Trace、RAG Eval、admin-only 观测入口 顺序完成。Trace schema、run_id、tool_calls、citations 和 eval 指标共享同一观测上下文。 Agent A/B/C 的 run、sources、quiz 输出稳定。 Trace 仅管理员可见；所有 prompt、token、provider secret、用户隐私字段已脱敏；eval 指标可追溯到 run 与 citation。
Agent F：安全与生命周期收口 删除、过期、限流、secret scan、发布前安全清单 最后收口，不提前并行改核心链路。该组横跨 DB、R2、Qdrant、Redis、API 和前端权限。 Agent A-E 的主要数据路径已经稳定。 删除 project 会清理 DB、R2、Qdrant、Redis；限流与 quota guard 生效；公开仓库不包含真实密钥、本地数据或缓存产物。

调度说明

  • 禁止强拆：同一个状态机、同一组表、同一条数据闭环、同一套 prompt 或 trace schema 被多个任务共同依赖时，默认给同一个 agent 顺序完成。
  • 允许并行：Agent A 完成 Parser + Index 后，Agent C 可启动 Quiz；Agent A 完成本地问答接口后，Agent D 可进入真实联调；Agent B 可在 Local Ask 稳定后接入 Web 扩展。
  • 交接要求：每个 agent 完成后必须更新相关 HTML 文档、记录测试命令、说明未实现逻辑和下一位 agent 的输入契约。

风险

  • 契约漂移：OpenAPI、Pydantic、SQL 和 TypeScript 必须同步修改。
  • 密钥泄露：真实 key 只能放在本地 .env 或云端 secret manager，不能进入仓库。
  • Trace 暴露：管理员 trace 不能出现在普通用户导航、公开路由或普通响应体中。
  • Prompt injection：上传文件和网页内容只能作为证据，不能作为系统指令。
  • Git 元数据缺失：当前目录没有暴露分支或提交历史，上传前需要重新确认提交范围。

停止与确认条件 Decision gates

  • 重命名 API 字段或响应 schema 前必须停下来确认。
  • 修改数据库 owner 字段、主键或删除策略前必须停下来确认。
  • 加入真实 provider key 或 signed URL 示例前必须停下来确认。
  • 让 Auto mode 默认调用 web 工具前必须停下来确认。
  • 把 Developer Trace 暴露给普通用户前必须停下来确认。
  • 把共享阻塞上下文拆给多个 agent 前必须停下来确认。
