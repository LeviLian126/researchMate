项目定位

这是工程作品集，而不是商业 SaaS

作品集目标

展示 AI 全栈架构、持久化工作流、检索质量、数据契约、可观测性、故障恢复、安全边界和 CI/CD，并为面试官提供可以沿代码和测试继续检查的证据。

明确不在目标内

获客、定价、订阅、支付、CRM、营销自动化和增长分析不是成功标准，也不会决定工程路线图。

阅读方式：招聘者可以快速判断项目范围；工程师可以沿契约和证据深入检查；演示用户验证的是系统行为，而不是商业转化漏斗。

支撑技术栈的核心场景

从来源集合到可审计的证据报告

普通文档聊天不足以证明这套技术栈的必要性。ResearchMate 的目标流程包含长时间摄取、并行证据工作、持久化人工决策、增量重计算和可度量质量。

01 / 摄取建立来源语料库上传数十到数百个文档与网页来源，异步完成解析、去重、版本管理、切块和索引。
02 / 规划拆解研究目标LangGraph 将宽泛问题分解为有边界的子问题，并保存检查点以支持中断和恢复。
03 / 提取构建证据模型Celery Worker 并行检索和提取主张，附加页级引用，并识别支持、冲突、重复与证据缺口。
04 / 审查处理不确定性低置信结论、不可信网页证据和提示词注入信号会暂停流程，等待明确的人工决定。
05 / 综合生成并更新报告生成带引用报告；语料库变化时，只重新计算受到影响的主张和章节。
06 / 证明评估系统比较检索和流水线版本，检查 Trace、延迟、Token、成本、重试、失败和恢复结果。

产品与路线图

用户工作流制造真实负载，工程实验室负责证明系统

研究工作区项目、来源库、Grounded Ask、主张、矛盾、人工审查和带引用报告。 证据审查 Agent有边界的 LangGraph 编排、并行工作、检查点、质量门和增量刷新。 评估实验室对检索、引用、忠实度、延迟和成本进行版本化数据集与实验比较。 可靠性实验室Trace 图、上下文健康度、故障注入、重试证据和 SLA 模拟。
依赖顺序

先确定契约，再完成持久化和认证，然后依次建设异步摄取、混合检索、Agent 工作流、评估、可观测性、MCP 和发布证据。

范围控制

不引入没有测量依据的 Kafka、Neo4j、第二套向量数据库、平行 Agent 框架或 Kubernetes 层。

架构与技术栈

前后端分离的模块化单体，结合异步任务和可恢复工作流

后端不是传统服务端渲染 MVC。FastAPI Router 负责 HTTP 接口，应用服务负责复杂用例编排，Repository Protocol 与 Provider 接口隔离 PostgreSQL、Qdrant、LLM、搜索和对象存储。

Next.js / Vercel作品集、研究工作区与工程实验室 UI。
FastAPI认证 REST 契约、权限检查和同步编排入口。
Celery Worker解析、切块、索引、Agent 工作流和评估任务。
Outbox Dispatcher将已提交的 PostgreSQL 意图可靠投递到 Redis。
LangGraph检查点、人工中断、恢复和有边界的 Agent 状态。
ProvidersNVIDIA、Tavily、Qdrant、Supabase 与 S3 兼容存储。
数据权威边界：PostgreSQL 保存不可丢失的业务事实；对象存储保存原始文件和解析产物；Qdrant 是可以重建的检索投影；Redis 只承担临时队列和协调，不保存不可恢复的业务真相。
架构分类

API-oriented modular monolith：分层架构与 Ports and Adapters 为主体，并结合 Transactional Outbox、后台 Worker 和工作流编排。

安全与可靠性

所有权和角色检查、RLS、统一错误体、请求 ID、幂等边界、Lease、重试、检查点和隐私安全遥测共同构成运行边界。

数据库与持久化契约

PostgreSQL 是业务事实源，其他存储承担明确的派生职责

15初始迁移表覆盖用户、项目、文档、任务、运行与基础业务状态。 16增量领域表覆盖工作流、证据、主张、报告、Outbox 和评估。
RLS租户隔离用户所有权既由应用层检查，也由数据库行级安全策略约束。
SHA迁移完整性有序且带校验和的迁移避免已发布数据库结构被静默改写。
持久化边界

Repository Protocol 使本地内存实现和 PostgreSQL 实现可以替换，上层 Router 与应用服务不依赖具体数据库驱动。

一致性边界

业务记录与 Outbox 意图在同一数据库事务中提交，再由独立 Dispatcher 投递异步任务，降低“数据库成功但消息丢失”的风险。

API 契约

35 个有类型的 REST 操作构成当前后端公共边界

接口覆盖身份、项目、文档、任务、Grounded Ask、来源与 Trace、证据审查、人工决定、报告、目录、评估、可靠性、健康检查与依赖就绪状态。

35REST 操作由 FastAPI/OpenAPI 生成并执行漂移检查。
1统一错误模型标准化错误代码、消息和 request_id，避免泄露堆栈。
JWT身份边界非本地环境要求有效 Supabase 会话，并执行所有权与角色检查。
MCP协议网关本地认证初始化已通过，消费者一致性仍待验证。
幂等性与状态冲突

创建型和长任务边界显式处理 Idempotency-Key、重复请求和状态冲突，并使用 409 等统一语义响应。

健康与就绪

Liveness 只证明进程存活；Dependency readiness 进一步检查数据库、Redis、检查点、Worker 心跳和 Outbox 健康度。
