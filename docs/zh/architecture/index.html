Runtime and recovery flows

Where complexity appears—and who owns failure

1 · Asynchronous document ingestion and deletion code implemented · integration unvalidated
Reserve + uploadFastAPI validates ownership, type, size, and quota; the S3-compatible adapter issues a short-lived content-type-bound signed URL and verifies HEAD metadata.
Commit + dispatchPostgres writes document, job, and outbox intent atomically; the SKIP LOCKED dispatcher publishes a deterministic Celery task ID with backoff and stale-claim recovery.
Parse + indexA lease-protected task checks bytes/checksum, preserves Docling provenance, replaces deterministic page/chunk rows, and upserts named dense+sparse Qdrant vectors.
Delete + convergeA deletion consumer removes Qdrant/object-storage projections before database children using deterministic identifiers and retry. Managed-service crash, poison-work, stale-vector, and partial-delete convergence tests remain.
Recovery truth: Postgres owns intent and lease state; workers own retryable processing. DOCX/PPTX smoke passed. PDF OCR/layout validation failed because its model download did not complete.
2 · Evidence Review Agent graph code + local fan-out smoke · durable runtime unvalidated
PlanThe LangGraph graph and SQL domain code decompose the goal and record graph/pipeline versions.
Retrieve + extractSend fan-out, selected-document Qdrant filters, optional Tavily retrieval, server-owned evidence IDs, and schema-validated claim extraction are implemented.
Budget + meterEvery model call atomically reserves a conservative ceiling, rejects oversized prompts, and records provider tokens, latency, and a configured-price estimate.
Reconcile + reviewRelation reconciliation and interrupt/resume decision code expose support, contradiction, duplication, gaps, and approve/edit/reject.
Resume + refreshAn expiring workflow lease rejects duplicate delivery; a persisted checkpoint resumes without replaying initial input. Changed-document refresh copies unaffected sections and transactionally publishes a new revision.
Unvalidated boundary: the installed LangGraph graph completes a bounded fake-domain fan-out smoke. No Postgres checkpoint table, interrupt/resume persistence, restart, Qdrant/provider execution, or managed incremental refresh has passed.
3 · Evaluation and reliability RAGAS import + budget/fault code · external execution unvalidated
ReserveBefore each case, the runner atomically reserves a conservative estimate against the run ceiling; this is a scheduler guard, not provider billing.
Execute + scoreLease-protected bounded execution writes deterministic metrics and, when available, RAGAS faithfulness scores.
Exercise failureAdmin-only local/test/preview commands create durable, expiring, non-destructive fault simulations with status polling.
Observe safelyOptional OTLP FastAPI spans, bounded JSON events, and Langfuse provider observations omit prompt/source/secret content.
Unvalidated boundary: RAGAS now imports under the pinned compatibility range, but no broker-backed fault run, provider score, OTLP export, Langfuse trace, accepted baseline, or cost reconciliation has passed.

Native Windows development

No local Docker requirement

Available now

Node.js 22.14.0, npm 11.6.0, Python 3.13.2, uv, Celery, Docling 2.111.0/core 2.86.0, and Git are installed on Windows under the project environment.

Installed AI runtime, bounded proof

LangGraph 1.2.9, the Postgres checkpoint package, RAGAS 0.4.3, MCP 1.28.1, OpenTelemetry, and Langfuse are installed locally. Local proofs cover LangGraph fan-out, RAGAS imports, strict serializer import, and MCP initialization; no real database, provider, collector, or Inspector flow has run.

Intentionally not required locally

PostgreSQL Server, Redis Server, Vercel CLI, GitHub CLI, and Docker are not prerequisites for application development.

Native application processes

Next.js, FastAPI, and the Windows-compatible Celery worker run as separate local processes using the existing Node and Python toolchains.

Managed development services

A separate Supabase Free development project is provisioned and healthy, with automatic public-table exposure disabled and automatic RLS enabled. Its selected no-card S3-compatible Storage path now has a private researchmate-dev bucket, a 25 MB PDF/DOCX/PPTX allowlist, and protocol enabled; no access key or request has been created. An isolated Upstash Redis Free database is also healthy in the same region with TLS enabled and zero use. A Qdrant Cloud Free development cluster is Healthy in its selected region, but contains no configured collection, application credential, or request evidence. Cloudflare R2 remains unactivated because its subscription can bill overages. No application secret, migration, database/broker/vector connection, or integration test has run. NVIDIA model/embedding endpoints, Tavily, an OTLP collector, and Langfuse are not configured.

Isolation rule: development and production use different projects, buckets, databases, vector clusters, API keys, data, and environment variables. Unit tests use fakes and isolated repositories; provider integration tests run only when explicitly enabled.
