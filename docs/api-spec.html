ResearchMate API Spec

TLDR: API 版本统一使用 /api/v1。所有业务接口默认需要 Bearer JWT。所有 project、document、run、quiz、trace 访问都必须做 user_id 归属校验。

1. 契约事实源

文件作用
infra/openapi/openapi.yaml and docs/openapi-spec.htmlHTTP API 的机器可读事实源
apps/api/src/researchmate_api/schemasFastAPI/Pydantic 请求与响应模型
packages/shared/src/contracts.ts前端可复用的 TypeScript 契约

2. 全局规则

2.1 认证

  • GET /api/v1/healthz 不需要认证。
  • 其他接口默认需要 Authorization: Bearer <token>。
  • 后端必须校验 token 的 issuer、audience、signature、expiry 和用户状态。

2.2 授权

所有资源接口必须验证：

request.user_id == resource.owner_user_id
管理员接口也必须校验角色，并记录审计日志。

2.3 错误响应

{
  "error": {
    "code": "DOCUMENT_NOT_READY",
    "message": "Document is still being processed.",
    "request_id": "req_..."
  }
}

错误响应不得包含：

  • SQL
  • stack trace
  • provider raw error
  • API key
  • OAuth token
  • R2 signed secret

5. 校验边界

5.1 请求校验

  • project_id、document_id、run_id、trace_id 必须是 UUID。
  • message 最大 8000 字符。
  • 上传文件最大 25MB。
  • 文件类型只允许 pdf、docx、pptx。
  • 搜索、抓取、Ask、Quiz 都要做 per-user 和 per-IP 限流。

5.2 输出校验

输出Schema必要校验
AskGroundedAnswerJSON parse、Pydantic、Source policy、Citation
QuizQuizSet题型、选项数量、答案、解析、来源摘要
TraceDeveloperTrace管理员权限、脱敏、无 secret

6. 前后端约定

  • 前端只渲染结构化字段，不解析模型原始文本。
  • Ask 顶部只显示 Mode 和 Sources 摘要。
  • Sources panel 使用 GET /runs/{run_id}/sources。
  • 普通用户 UI 不显示 trace、prompt、token 明细和工具 raw payload。
  • Admin Trace 没有普通导航入口。

7. 非实现说明

当前 apps/api 只保留路由和 schema。业务逻辑未实现时统一返回 501，这是刻意边界，不是 bug。

# ResearchMate API Spec

> **TLDR:** API 版本统一使用 `/api/v1`。所有业务接口默认需要 Bearer JWT。所有 project、document、run、quiz、trace 访问都必须做 `user_id` 归属校验。

## 1. 契约事实源

| 文件 | 作用 |
|---|---|
| `infra/openapi/openapi.yaml and docs/openapi-spec.html` | HTTP API 的机器可读事实源 |
| `apps/api/src/researchmate_api/schemas` | FastAPI/Pydantic 请求与响应模型 |
| `packages/shared/src/contracts.ts` | 前端可复用的 TypeScript 契约 |

## 2. 全局规则

### 2.1 认证

- `GET /api/v1/healthz` 不需要认证。
- 其他接口默认需要 `Authorization: Bearer <token>`。

所有资源接口必须验证：

```text
request.user_id == resource.owner_user_id
```

> 管理员接口也必须校验角色，并记录审计日志。

### 2.3 错误响应

```json
{
  "error": {
    "code": "DOCUMENT_NOT_READY",
    "message": "Document is still being processed.",
```

错误响应不得包含：

- SQL
- stack trace

## 4. Source Mode 与 Task Type
