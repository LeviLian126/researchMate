API Spec

定义当前已实现的本地 MVP API、鉴权边界、错误模型和可替换云端 adapter 边界。

TLDR: API 版本统一使用 /api/v1。除 /healthz 外，所有业务接口默认需要 Bearer token；所有 project、document、run、quiz、trace 访问都按 owner user_id 校验。

当前实现状态

模块状态说明
Auth已实现本地 dev tokendev、dev-admin、dev-user-a、dev-user-b 用于本地联调；上线替换为 Supabase JWT 校验。
Project已实现创建、列表、读取、删除均做用户隔离。
Document已实现本地上传闭环upload-url 返回 R2 signed URL 占位；complete 支持 extracted_text 模拟 worker 解析并写入 chunks。
Ask已实现支持 /study、/search、/hybrid 与 mode selector；Auto 默认 local-only。
Quiz已实现基于本地 chunks 生成题目；选择题恰好四个选项；题目保存历史。
Sources已实现按 run_id 展开 citations 与来源统计。
Developer Trace已实现 admin/developer-only普通用户不可见；trace 仅记录脱敏计划、工具摘要、检索片段和校验结果。
Web provideradapter 待接真实 API本地使用 demo evidence，不联网、不需要真实 SERPER/JINA/FIRECRAWL key。

Source Mode 与 Slash Command

  • auto 默认解析为 local_only，控制成本、延迟和隐私。
  • /study 强制本地资料问答；没有 ready chunks 时返回 DOCUMENT_NOT_INDEXED，不编造答案。
  • /search 强制 web-only；当前本地运行只返回 demo evidence，上线后替换真实搜索和抓取 provider。
  • /hybrid 先本地检索，本地为空时可触发 web fallback。
  • /quiz 走 POST /api/v1/quiz，不允许混入 Ask route。

统一错误体

{
  "error": {
    "code": "DOCUMENT_NOT_INDEXED",
    "message": "No ready local document chunks exist for this project.",
    "request_id": "req_local_dev"
  }
}

输出与安全约束

  • 错误响应不得包含 SQL、stack trace、provider raw error、API key、OAuth token 或 R2 signed secret。
  • Developer Trace 不进入普通用户导航；普通 user token 调用返回 403 ADMIN_REQUIRED。
  • 前端只读取 NEXT_PUBLIC_API_BASE_URL 等 public 配置，不包含后端密钥名或密钥值。
  • 机器可读事实源为 docs/openapi-spec.html 与 infra/openapi/openapi.yaml。
