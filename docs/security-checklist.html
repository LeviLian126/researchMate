Security Checklist

上线前安全边界、已落地控制、剩余 secret/provider adapter 检查和冻结条件。

TLDR: 当前仓库不包含真实密钥；本地 API 已实现用户隔离、MIME 校验、错误脱敏、quota guard、source-policy guard 和 admin-only Developer Trace。上线前必须把本地 adapter 替换为云端安全实现。

控制项状态实现位置

Secret 规则

  • 真实 SUPABASE_SERVICE_ROLE_KEY、R2_SECRET_ACCESS_KEY、NVIDIA_API_KEY、SERPER_API_KEY、JINA_API_KEY、FIRECRAWL_API_KEY 只能进入本地 .env 或部署平台 secret manager。
  • 前端只能使用 NEXT_PUBLIC_* 配置，不读取 service role、provider key 或 R2 secret。
  • 日志、错误响应、Trace、Sources panel 不得显示 token、signed secret、raw provider payload。

数据生命周期

  • Project delete 当前会在本地 store 标记删除并清理关联 chunks。
  • Document delete 当前会标记删除并清理关联 chunks。
  • 上线时必须把删除动作扩展到 Supabase、R2、Qdrant、Redis cache 和 job queue。
  • 过期策略以 expires_at 为统一字段；worker 应执行定期清理。

Provider 安全

Provider上线前要求
LLM强制 JSON/schema validation；失败重试不得扩大来源权限；prompt 不记录 secret。
Embedding只发送必要 chunk text；payload filter 必须包含 user_id/project_id。
Search/Crawl记录 URL、时间戳、缓存状态；失败网页不得进入 citation。
StorageR2 key 必须带 user_id/project_id/document_id；signed URL 短时有效。

冻结条件

  • 测试未通过、handoff dashboard validation 未通过，不进入上线。
  • 发现真实 secret、私有缓存、node_modules、.venv、__pycache__ 被打包，立即停止。
  • 任何普通用户可见 Trace、stack trace、raw provider error 或其他用户数据，立即停止。
  • Auto 默认联网、Local Ask 调用 web 工具或 citation 无来源，立即停止。
