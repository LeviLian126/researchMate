Next actions

下一步顺序

  1. 先完成生产 Auth：Supabase Google/GitHub OAuth、JWT issuer/audience/signature/expiry 校验，并让前端按真实角色隐藏 Trace。
  2. 再替换数据层：Supabase Postgres/RLS、Cloudflare R2 signed URL、Qdrant 向量检索、Redis job/cache/rate limit。
  3. 接入真实 AI 与 Web provider：LLM、embedding、Serper/Tavily、Jina/Firecrawl，并保留 source policy 和 citation validation。
  4. 最后做上线：部署 Frontend/API/Worker，填入 GitHub Secrets 或平台 Secret Manager，跑生产冒烟和 canary。

Decision gates

停止条件

  • 任何真实 API key、OAuth token、R2 secret、provider secret 准备进入仓库时，必须停止。
  • 要把 Auto 默认改成联网时，必须先做隐私、成本和安全评审。
  • 要让普通用户看到 Developer Trace 入口时，必须先重做权限与前端可见性设计。
  • 修改 owner isolation、RLS、signed URL 或删除逻辑时，必须先补隔离测试。
