Conventions

Transport, identity, and ownership

Authentication

GET /api/v1/healthz and GET /api/v1/readyz are public for platform probes. Every other REST operation declares HTTP bearer authentication. Local/test mode accepts only named or explicit dev: identities; unknown tokens fail closed. Preview/production configuration requires Supabase JWT mode, whose JWKS signature, issuer, audience, expiry, subject, and role mapping are implemented but not provider-validated.

Ownership

Handlers resolve a CurrentUser; Store lookups scope resources to that user. A missing resource and another user's resource both return 404 to avoid disclosing identifiers.

Error envelope

Runtime failures use {"error":{"code","message","request_id"}}. Request validation is normalized to VALIDATION_FAILED; the same validated/generated request ID is returned in X-Request-ID.

Known machine/runtime drift. OpenAPI still declares generated HTTPValidationError bodies for some 422 responses while runtime handlers normalize validation to the project error envelope. Ask and Quiz can also return runtime 409 and 429 responses not present in their OpenAPI response maps.

GET /api/v1/readyz — check non-charging dependency readiness
Caller / access
Deployment platform, readiness smoke, or operator. Public; no bearer token and no private payload.
Input
No path, query, or body fields.
Success
200 with status:"ready", environment, component states, and an empty failed-component list. Local/memory dependencies may be not_required.
Failure / semantics
503 with status:"not_ready" and safe failed component names. It executes select 1 for Postgres, verifies the four SDK-owned checkpoint tables, checks fresh worker/dispatcher heartbeats, rejects stale or exhausted outbox work, pings Redis, and reads Qdrant collection metadata. S3-compatible object storage, LLM, and web search remain configuration checks only, so it never spends NVIDIA or Tavily quota.
Current boundary
Route and deterministic local/memory tests exist. No managed Postgres/Redis/Qdrant/object-storage/NVIDIA/Tavily readiness response or background heartbeat has been validated.

Implemented contract evolution

Research, decision, evidence, report, evaluation, reliability, and fault operations

The 16 advanced REST operations below are present in FastAPI and generated OpenAPI. Together with the original 18 and readiness, the REST total is 35. Their route, schema, ownership/role, idempotency, and deterministic repository behavior have code evidence; managed providers, queue delivery, checkpoint recovery, telemetry export, and deployment remain integration-unvalidated. MCP is a separate protocol boundary and is not counted as REST.

Shared conventions: bearer authentication; owner/project permission checks in the API; Idempotency-Key on commands; the current error envelope with a request ID; durable Postgres operation truth; persisted SSE replay; and Redis as delivery rather than irreplaceable state. Supabase JWT, managed Postgres, and distributed delivery are not yet validated.
How to read the rows: request and success shapes describe the implemented route schemas. Local LangGraph, RAGAS, strict-checkpoint serializer, and MCP initialization tests now pass. Failure/recovery and acceptance rows still name distributed-system gates; they are not evidence that a managed checkpoint, provider, queue, or cloud runtime passed.
POST /api/v1/research-runs — start an evidence review implemented contract · runtime unvalidated
Caller / access
Authenticated project owner. The route verifies project, accepted pipeline, and every explicitly selected ready document. Daily and concurrent-run quotas remain unvalidated; the worker enforces the per-run cost ceiling before each model call.
Request
{ project_id: UUID, research_goal: string 20..12000, source_scope: {document_ids?: UUID[], allow_web: boolean}, pipeline_version_id: UUID, review_policy: "strict"|"balanced", max_cost_usd?: decimal }; required Idempotency-Key.
Success / async
202 { run_id, status:"pending", status_url, events_url, created_at }. Postgres commits run and outbox intent before queue publication.
Failure / recovery
404 PROJECT_NOT_FOUND; 409 PIPELINE_NOT_ACCEPTED, SOURCE_DOCUMENT_NOT_READY, SOURCE_SCOPE_EMPTY, or duplicate key with mismatched body; request validation rejects duplicate document IDs. During execution WORKFLOW_BUDGET_EXCEEDED, invalid pipeline configuration, or provider failures produce a durable failed run. A matching command duplicate returns the original operation.
Acceptance
One logical run is created for repeated delivery; owner isolation holds; duplicate task delivery acquires at most one expiring workflow lease; an existing checkpoint resumes without replaying initial input; outbox recovery dispatches after an API crash.

POST /api/v1/reports/{report_id}/refresh — recompute impacted sections incremental code · LangGraph unvalidated
Caller / access
Report owner with refresh and concurrent-run quota.
Request
{ changed_document_ids?: UUID[], force_sections?: string[], pipeline_version_id: UUID }; required Idempotency-Key.
Success / async
202 with refresh run ID, base revision, planned revision, impacted section keys, and event URL.
Current implementation
The repository validates changed documents as owned/ready, derives directly affected section keys from stored citation edges, and falls back to all existing sections for a newly added document with no prior edge. Worker code restricts Qdrant retrieval to changed document IDs, requires exact generated impacted keys, copies unaffected sections unchanged, and invalidates the prior revision in the same final transaction.
Failure / recovery
404 REPORT_NOT_FOUND; 409 REPORT_ALREADY_REFRESHING or pipeline mismatch; invalid documents/sections return domain errors. Failed refresh is designed to leave the published revision unchanged, but this and restart behavior are runtime-unvalidated.
Acceptance
Impact/filter/preservation code exists; it still needs a real LangGraph/Postgres/Qdrant execution proving only affected work changes and the previous report survives any failure.
POST /api/v1/evaluation-runs — launch a versioned experiment implemented contract · RAGAS import tested
