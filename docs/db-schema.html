3. 安全边界

3.1 用户隔离

所有业务查询必须带上：

where user_id = :current_user_id

如果表只有 project_id，必须先由 projects.user_id 解析 owner。

3.2 Row Level Security

迁移中已给核心 owner 表加 RLS 草案：

  • projects
  • documents
  • chunks
  • messages
  • ask_runs
  • quiz_sets
  • jobs
  • deletion_jobs
  • api_usage
后端仍然必须做权限校验。RLS 是第二道防线，不是替代品。

3.3 Trace 脱敏

tool_calls.input 可以存结构化摘要，但不能存：

  • API key
  • OAuth token
  • R2 signed secret
  • 完整 system prompt
  • 原始 provider secret

# ResearchMate DB Schema

> **TLDR:** Postgres 保存业务元数据和引用关系，Qdrant 只保存向量与检索 payload。所有用户数据表都必须能通过 `user_id` 或 `project_id` 找回 owner。

## 1. 数据库事实源

| 文件 | 作用 |
|---|---|
| `infra/supabase/migrations/202605260001_initial_schema.sql` | Supabase Postgres 初始迁移 |
| `infra/qdrant/researchmate_chunks.json` | Qdrant collection payload 约束 |
| `docs/db-schema.html` | 人类可读说明 |

## 2. 核心实体

### 2.1 用户与项目

| 表 | 作用 | owner 字段 |
|---|---|---|
| `profiles` | 用户资料和角色 | `id` |
| `projects` | 用户项目 | `user_id` |
| `conversations` | Ask 会话 | `user_id`, `project_id` |
| `messages` | 用户和助手消息 | `user_id`, `project_id` |

### 2.2 文档与检索

| 表 | 作用 | 关键字段 |
|---|---|---|
| `documents` | 文件元数据 | `status`, `r2_object_key`, `expires_at` |
| `document_pages` | 页、slide、section 文本 | `page_no`, `slide_no`, `metadata` |
| `chunks` | chunk 元数据 | `source_type`, `qdrant_point_id`, `expires_at` |

### 2.3 执行与输出

| 表 | 作用 |
|---|---|
| `ask_runs` | 一次 Ask 或 Quiz 执行 |
| `tool_calls` | 工具调用摘要 |
| `citations` | claim 到 chunk/page/url 的引用 |
| `quiz_sets` | 一次测验批次 |
| `quiz_questions` | 题目、答案、解析、来源 |

### 2.4 运维与生命周期

| 表 | 作用 |
|---|---|
| `jobs` | 解析、索引、抓取、eval 等异步任务 |
| `deletion_jobs` | 删除 DB/R2/Qdrant/Redis 的可重试任务 |
| `api_usage` | 限流和成本统计 |

## 3. 安全边界

### 3.1 用户隔离

所有业务查询必须带上：

```sql
where user_id = :current_user_id
```

如果表只有 `project_id`，必须先由 `projects.user_id` 解析 owner。

### 3.2 Row Level Security

迁移中已给核心 owner 表加 RLS 草案：

- `projects`
- `documents`
- `chunks`
- `messages`
- `ask_runs`
- `quiz_sets`
- `jobs`
- `deletion_jobs`
- `api_usage`

> 后端仍然必须做权限校验。RLS 是第二道防线，不是替代品。

### 3.3 Trace 脱敏

`tool_calls.input` 可以存结构化摘要，但不能存：

- API key
- OAuth token
- R2 signed secret
- 完整 system prompt
- 原始 provider secret

## 4. 状态机

### 4.1 Document status

```text
uploaded -> parsing -> parsed -> indexing -> ready
                         |
                         v
                       failed
```

删除或过期：

```text
ready -> deleted
ready -> expired -> deletion_jobs
```

### 4.2 Job status

```text
pending -> running -> succeeded
pending -> running -> failed
pending -> cancelled
```

## 5. Qdrant Payload

Qdrant collection: `researchmate_chunks`

每次检索必须带：

```json
{
  "user_id": "current-user",
  "project_id": "current-project",
  "source_type": "local_doc"
}
```

Hybrid 检索可分两次执行：

1. `source_type = local_doc`
2. 必要时 `source_type = web_page`

## 6. 删除范围

删除项目必须覆盖：

- `projects`
- `documents`
- `document_pages`
- `chunks`
- `messages`
- `ask_runs`
- `tool_calls`
- `citations`
- `quiz_sets`
- `quiz_questions`
- R2 raw file 和 parsed asset
- Qdrant points
- Redis cache/job status

## 7. 验收标准

- [x] 表结构覆盖 Ask、Quiz、Sources、Trace、Deletion。
- [x] 所有用户数据表可通过 `user_id` 隔离。
- [x] Qdrant payload 足以做 metadata filter。
- [x] 删除任务有独立表，失败可重试。
- [x] `.env.example` 不包含真实 secret。
