Test Plan

定义当前本地 MVP 的测试范围、验收命令、已覆盖场景和上线前补充检查。

TLDR: 当前验收以 pytest 为事实源，覆盖仓库契约、API 业务闭环、用户隔离、Quiz、Trace 权限、MIME 安全和前端页面契约；Node 依赖未安装时不声称通过 Next build。

验收命令

命令目的通过标准
pytest tests/test_project_scaffold.py tests/test_api_workflow.py tests/test_frontend_contracts.py -q主测试套件全部通过。
python skill/agent-context-html/scripts/validate_context_dashboard.py docs/handoff验证 Agent Dashboard输出 PASS。
ruff check apps/api/src workers/ai-worker/src testsPython lint环境安装 ruff 后应通过；当前容器若缺 ruff，不把它记为通过。
npm --workspace @researchmate/web run buildNext.js 构建安装 node_modules 后应通过；当前交付不包含依赖目录。

已覆盖场景

  • Project 创建、读取、列表、删除按 user_id 隔离。
  • Document 上传地址生成、MIME 类型校验、上传完成、解析切片和 job 状态。
  • Local Ask 在无 ready chunks 时返回 DOCUMENT_NOT_INDEXED，不编造答案。
  • Local Ask 返回 citations、Sources panel 与 Developer Trace。
  • 普通用户无法读取 Developer Trace；developer/admin 可读取脱敏 trace。
  • Quiz 生成、历史列表、引用绑定、选择题四选项约束。
  • Source Policy validator 阻止 local-only 调用 web 工具。
  • 前端存在 Project、Ask、Library、Quiz、Trace 页面，且不携带后端密钥名。

手动冒烟脚本

  1. 启动 API 与 Web。
  2. 使用 dev-user-a 创建项目并上传 PDF/DOCX/PPTX 文件，在 textarea 中填写可解析文本。
  3. 在 Ask 页面使用 /study explain RAG，检查 answer、citations、sources panel。
  4. 切换 dev-user-b，确认看不到用户 A 的项目或文档。
  5. 生成 Quiz，确认选择题恰好 4 个选项且有 source citation。
  6. 用普通 user token 打开 trace 链接应失败；用 dev 或 dev-admin 可查看脱敏 trace。
