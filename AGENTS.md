# ResearchMate Agent Instructions

## TLDR

- 使用简单清晰的语言准确说明代码与功能边界。
- 禁止使用意义不明的简写。
- 修改后的文件路径向用户展示时使用相对路径。
- 写 Markdown 文档时使用标题、加粗、引用、表格等格式提升可读性。
- 写类、函数等代码块时，每个类或函数前添加一行简短注释说明作用。

## Current Project Boundary

ResearchMate 当前阶段只做项目初始化、API 契约、数据库结构、校验边界、任务拆解和 handoff 页面。

> 不写完整业务逻辑，不接真实 provider，不把真实密钥放进仓库。

## Documentation Rule

`docs` 目录内的项目文档使用 HTML 页面维护，不再新增 Markdown 文档。
